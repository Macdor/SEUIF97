# SEUIF97 软件设计说明文档

## 1. 文档概述

### 1.1 文档目的
本文档为水和水蒸汽物性高速计算库 SEUIF97 的软件设计说明，详细描述系统架构、模块划分、接口定义和算法实现等核心设计要素，为后续维护、扩展和二次开发提供完整的技术参考。

### 1.2 适用范围
本文档适用于 SEUIF97 项目的核心 C 实现代码，覆盖所有水和蒸汽热力性质计算、传输性质计算和汽轮机热力过程计算功能模块。

### 1.3 术语定义
| 术语 | 定义 |
|-----|------|
| IAPWS-IF97 | 国际水和水蒸汽性质协会 1997 年发布的工业用热力学性质公式标准 |
| Region 1 | 亚临界水区 |
| Region 2 | 过热蒸汽区 |
| Region 3 | 临界区附近 |
| Region 4 | 饱和线区域 |
| Region 5 | 高温区（800℃以上） |
| 最短加法链 | 用于整数幂次快速计算的最优乘法序列算法 |
| 幂缓存 | 预计算并缓存不同幂次以避免重复多项式计算 |

## 2. 系统概述

### 2.1 系统目标
SEUIF97 是一个高性能的 C 语言实现的 IAPWS-IF97 标准计算库，专为计算密集型场景（如非稳态过程模拟、在线过程监控与优化）设计，其计算速度显著超越标准库的 math.pow() 和重复平方法，同时保持完全的 IAPWS-IF97 精度。

### 2.2 主要特点
1. 完全遵循 IAPWS-IF97-Rev 国际工业标准
2. 独创三大核心加速算法
3. 跨平台支持 Windows、Linux 操作系统
4. 提供 11 种编程语言接口支持
5. 支持 12 种输入状态对组合，计算 36 种热力学及传输性质

### 2.3 性能指标
- 计算速度：比标准 C 库的 math.pow() 快数倍
- 精度：完全符合 IAPWS-IF97 标准精度要求
- 并发安全：纯数学计算，无全局可变状态
- 内存占用：极低内存占用，适合嵌入式和高性能场景

## 3. 系统架构设计

### 3.1 总体分层架构
SEUIF97 采用清晰的模块化分层架构，分为 6 大核心子系统：

```
┌──────────────────────────────────────────────────────────────┐
│                      对外接口层 (seuif97.h)                 │
│  pt, ph, ps, pv, th, ts, tv, hs, px, tx, hx, sx, ishd, ief  │
├──────────────────────────────────────────────────────────────┤
│                      公共功能层 (common/)                    │
│  区域判断、边界方程、物性对分发、传输性质、热力学过程         │
├──────────────────────────────────────────────────────────────┤
│                      加速算法层 (algo/)                      │
│  最短加法链、功率缓存、多项式递推、弦线法求根                 │
├──────────────────────────────────────────────────────────────┤
│         区域实现层 (r1/, r2/, r3/, r4/, r5/)                │
│  5 个独立 IAPWS-IF97 区域的方程实现与多项式计算              │
└──────────────────────────────────────────────────────────────┘
```

## 4. 模块详细设计

### 4.1 common 公共功能模块

#### 4.1.1 模块职责
- 统一定义输出物性 ID 枚举
- 提供跨区域的区域判断函数
- 实现边界方程求解
- 分发 12 种物性对到对应区域实现
- 传输性质（粘度、导热系数等）计算
- 汽轮机热力过程计算接口

#### 4.1.2 核心头文件
- `seuif97.h`：对外公共 API 声明
- `property_id.h`：物性 ID 常量定义（0-29）
- `common.h`：通用数据结构和函数指针类型定义
- `constant.h`：物理常量定义

#### 4.1.3 主要功能列表
| 文件名 | 功能 |
|-------|------|
| seuif97.c | 顶层 API 入口分发 |
| property_pair.c | 12 种输入状态对的通用分发逻辑 |
| region.c | p/T, p/h, p/s, h/s 等输入对的区域判断 |
| boundaries.c | 区域边界方程实现（如 B23 边界） |
| transport.c | 动态粘度、运动粘度、导热系数等传输性质 |
| thermodynamic_process.c | 汽轮机等熵焓降、等熵效率计算 |

### 4.2 algo 加速算法模块

#### 4.2.1 模块职责
本模块是 SEUIF97 的核心创新所在，实现三大独创加速算法。

#### 4.2.2 核心算法实现

**1. 最短加法链算法 (sac.c)**
- 算法思想：为给定整数 n 生成最短加法链序列，通过最少的乘法次数计算 x^n
- 文件：`sac.c`
- 函数：`ipowsac(double x, int n)`
- 支持幂次范围：0-58

**2. 功率缓存策略 (polynomial_solo.c)**
- 算法思想：预计算并缓存所有不同的指数幂，避免在多项式循环中重复计算 x^i
- 关键优化：从多项式的 n_i π^I_i η^J_i 中提取所有不同的 I 和 J，仅计算这些不同幂次
- 数据结构：`soI_pow[]`、`soJ_pow[]` 为幂次缓存数组
- 函数指针：`solo_power_fn` 用于幂次预计算回调

**3. 多多项式递推法 (polynomial_solo.c)**
- 算法思想：先计算基多项式（零阶），然后通过导数关系直接递推一阶、二阶混合偏导多项式，最大化共享计算结果
- 典型实现：`polys_solo_i_ii_ij_jj()` 一次计算同时返回 6 个多项式结果

**4. 弦线法求根 (root.c)**
- 用于反向方程求解（如 T(p, h) 等）
- 函数：`rtsec1()`, `rtsec2()`
- 精度：`xacc = 1.0E-08`
- 最大迭代次数：100 次

**5. 传统快速幂对比实现 (rqm.c)**
- ipowrqm()：快速幂方法，用于性能对比和调试

### 4.3 区域实现模块 (r1-r5)

每个区域采用完全相同的架构模式，保证代码一致性和可维护性：

| 区域 | 物理意义 | 核心特征方程 |
|-----|---------|-------------|
| r1 | 亚临界水区 | 吉布斯自由能方程 g(π, τ) |
| r2 | 过热蒸汽区 | 吉布斯自由能方程 g(π, τ) |
| r3 | 临界区附近 | Helmholtz 自由能方程 |
| r4 | 饱和线区 | p/T → 饱和温压关系，湿蒸汽性质 |
| r5 | 高温区 (>800℃) | 吉布斯自由能方程 g(π, τ) |

每个区域子目录包含：
- `regionX.h`：该区域的头文件声明
- `regionX_coff.h`：硬编码的多项式系数表
- `regionX_solo_ij.h`：幂次索引映射数组
- `regionX_solo_power.c`：该区域专属的幂次预计算逻辑
- `regionX_gfe.c / regionX_hfe.c`：基本特征方程
- `regionX_pT.c`：pT 输入对的物性计算
- `regionX_pT_ext.c`：pT 扩展物性（导数、传输性质等）
- `regionX_p_hs.c`：反向方程 (p, h), (p, s), (h, s)
- `regionX_T_phps.c`：温度反向方程 T(p, h), T(p, s)
- `regionX_pair.ext.c`：扩展物性对分发

## 6. 数据结构设计

### 6.1 核心数据结构定义

在 `src/common/common.h` 中定义的基础数据结构：

```c
// 多项式系数结构：存储 (I, J, n) 三元组
typedef struct {
  int I, J;     // 两个无量纲变量 π 和 τ 的幂次
  double n;     // 多项式系数
} IJnData;

// 区域属性计算函数指针类型
typedef double (*properties_region)(double, double, int);

// 区域判断函数指针类型
typedef int (*region_fn)(double, double);

// 通用物性计算函数指针类型
typedef double (*prop_fn)(double, double, int o_id);

// 弦线法求根目标函数指针类型
typedef double (*callfunc)(double, double);

// 快速缓存幂次预计算函数指针类型
typedef void (*solo_power_fn)(double, double, double *, double *);
```

## 7. 构建系统设计

### 7.1 支持的构建方式

SEUIF97 提供两种主流构建方式，完全跨平台：

#### 方式一：Makefile 构建（MinGW-GCC / GCC）
```makefile
make          # 直接编译生成动态库
make tests    # 编译所有单元测试程序
```
- Windows 下生成：`./bin/libseuif97.dll`
- Linux 下生成：`./bin/libseuif97.so`

#### 方式二：CMake 构建（支持 MSVC, GCC, Clang 等所有编译器）
```bash
cmake -B ./build/
cmake --build ./build/ --config Release
```

### 7.2 编译优化选项
在 Makefile 中使用 `-O3` 最高优化级别，保证计算性能最大化。
同时支持 `BUILD_DLL` 宏定义，生成兼容 32 位/64 位 VBA 的 `__stdcall` 调用约定导出。

### 7.3 MSVC 特殊处理
在 CMakeLists.txt 中设置 `CMAKE_WINDOWS_EXPORT_ALL_SYMBOLS ON`，自动导出所有函数符号，无需手动编写 .def 模块定义文件。

## 9. 性能设计要点

1. **-O3 最高优化级别**：编译器自动执行循环展开、指令调度等深度优化
2. **无全局可变状态**：所有计算函数为纯函数，线程 100% 安全
3. **栈上局部变量**：不使用动态内存分配（mallc/free），零 GC 停顿
4. **多项式循环优化**：通过三大加速算法大幅减少乘法运算次数
5. **无冗余分支判断**：区域判断仅执行一次，后续所有物性计算复用区域结果
